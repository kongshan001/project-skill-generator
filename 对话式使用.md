# Project Skill Generator - 对话式使用快速指南 🚀

**最重要的事：用自然语言就行，不需要命令行！**

---

## 💬 对话式使用（推荐）

### 第一步：告诉我要做什么

```
你: 帮我分析项目并生成技能文档
```

就这样！Claude Code 会自动：
1. 扫描你的项目
2. 识别模块
3. 生成技能文档

---

## 🎯 常用对话指令

### 分析和生成

```
"分析项目结构"
"生成技能文档"
"为新模块生成技能"
"配置专家团队"
```

### 更新和维护

```
"更新技能文档"
"同步最近的代码变更"
"重新生成 [模块名] 的技能"
```

### 查询和理解

```
"项目有哪些模块？"
"[模块名] 的主要功能是什么？"
"模块之间的依赖关系？"
```

### 专家咨询

```
"@backend-expert 如何优化API性能？"
"@database-expert 数据库查询太慢，怎么办？"
"我需要哪些专家配合？"
```

## ✨ 完整示例

### 场景 1：首次使用

```
你: 帮我分析这个项目

Claude: 好的！正在分析...

发现：
- 8个模块
- FastAPI项目
- 约5,000行代码

需要我生成技能文档和专家配置吗？

你: 好的

Claude: 开始生成...
✅ 8个技能文档
✅ 3个专家配置
完成！
```

### 场景 2：添加新功能

```
你: 我添加了一个支付模块，生成技能

Claude: 扫描支付模块...
发现：stripe_client.py, payment_processor.py

生成技能文档...
✅ 已生成 payment/SKILL.md

要更新专家团队吗？

你: 更新一下

Claude: 添加 Payment Expert...
✅ 已更新团队配置
```

### 场景 3：代码重构后

```
你: 我重构了认证模块，更新技能

Claude: 对比变更...
发现：新增2FA支持，修改了API

更新技能文档...
✅ 保留你的自定义注释
✅ 更新API文档
✅ 记录变更日志
```
